Isn't Everything an Object?

Take a look at these two snippets of code

Here's Javascript


var dictionario = {
	"aardvarkio": "a large African animal that has a long nose and has an 'io' at the end",
	"bananio": "a long curved fruit with a thick peel that is yellow and has an 'io' at the end.",
	"cougario": "an attractive woman in her 30's or 40's who is on the hunt once again and has an 'io' at the end"
}

Now here's Ruby:


dictionario = {
	aardvarkio: "a large African animal that has a long nose and has an 'io' at the end.",
	bananio: "a long curved fruit with a thick peel that is yellow and has an 'io' at the end.",
	cougario: "an attractive woman in her 30's or 40's who is on the hunt once again and has an 'io' at the end."
}

Seeing double? These bits of code are almost written in almost the exact same but in practice they perform slightly differently in their respective languages.

The above Javascript snippet is an example of a literal notation object. This acts as a repository for information which can be accessed and added to without using predefined methods. The two forms of notation for accessing values JS objects look like this:


dictionario[aardvarkio]
// and
dictionary.aardvarkio

You can even alter the selected values by setting either of them equal to a new value. Now let's take a look at accessing hash values in Ruby. The Ruby example near the top of this post is an example of a hash, another data storage method in Ruby. Unlike Javascript literal notation objects like the ones listed above, these cannot be accessed and added with a dot notation - so dictionary.aardvarkio is out of the question:


dictionario[:aardvarkio]

This is still kind of similar, but not as flexible as the syntax with the Javascript object. This is because JavaScript objects can be used in a way similar to Ruby classes. There is another type of notation for objects called a constructor notation which I will cover in a future blog post. For now, sit and ponder the amazing translation of syntaxes from one to another and remember that between Ruby and Javascript, there are many subtle differences that will become apparent over time.
